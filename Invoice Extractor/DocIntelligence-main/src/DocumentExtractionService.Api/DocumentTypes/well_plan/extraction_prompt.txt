STEP 1 — IDENTIFY THE PAD/PROGRAM AND RIG

STEP 2 — EXTRACT EACH WELL'S TECHNICAL DATA
══════════════════════════════════════════════════════════════

FOR MULTI-WELL PADS (batch_drilling_program, drilling_procedure):
Find each well's Technical Summary or Well Info page:
- Well name, coordinates, total depth (MD & TVD)
- Ground Level (GL) and Kelly Bushing (RKB/KB)
- Identifiers: API#, AFE, operator well IDs, permit numbers

From montage diagrams or well schematics:
- Formation tops: formation NAME + MD + TVD for each formation
- Casing program: section name, hole size, OD, ID, grade, weight, connection, depth range
- Cement details: type, density, volume details
- Drilling fluid: type, weight range
- FIT/LOT pressures

IMPORTANT FOR TABULAR FORMATS (like ConocoPhillips Drilling Procedures):
- Well data is in WIDE TABLES with 4-5 wells as COLUMNS
- Read COLUMN by COLUMN, not row by row
- Each column is one well — do not mix data between columns
- Data may span multiple pages — wells 1-4 on one page, wells 5-9 on next

FOR SINGLE-PAGE DRILLING OUTLINES:
- ALL data is on one dense page — use the PAGE IMAGE as primary source
- Formation tops may be in a small column/table on the left
- Casing is shown alongside the wellpath diagram
- Mud properties and BHA are in notes sections
- Risks may be bullet points in a corner

FOR SINGLE WELL PROGRAMS (Saudi Aramco, YPF):
- Well schematic on a dedicated page ("Esquema de pozo", "Diagrama de pozo")
- Formation column ("Columna litológica", "Pases formacionales")
- Casing design may be a separate section
- Drilling parameters per section in dedicated tables
- Spanish sections: "Sección Guía" → Surface, "Intermedia" → Intermediate

FOR PRE-SPUD MEETINGS (presentations):
- Extract available data from slides (well names, drilling order, basic well info)
- Set empty arrays [] for casing_program, formation_tops, drilling_sections, drilling_fluids
- Set confidence scores LOW (0.2-0.4)
