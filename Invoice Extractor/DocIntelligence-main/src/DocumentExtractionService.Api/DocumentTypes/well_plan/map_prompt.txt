══════════════════════════════════════════════════════════════
  STEP 2 — FIND EVERY WELL
══════════════════════════════════════════════════════════════

Scan the [PAGE N of M] markers throughout the text. For each well:

WELL NAMES appear in:
- Cover page well lists (often with API/AFE numbers)
- Table column headers in multi-well pads
- Section headers: "Well: UNIT_27_BD_512H"
- Schematic titles: "Wellbore Diagram — 512H"
- Footer labels on per-well pages

PAGE ASSIGNMENT RULES:
- A page belongs to a well if it contains ANY data for that well
- Shared pages (pad summary with all wells) → assign to ALL wells
- Cover page → assign to ALL wells
- Multi-well table pages → assign to each well that appears as a column
- Per-well technical pages → assign to that specific well
- Appendices with well-specific data → assign to the relevant well
- General appendices (rig inventory, safety) → do NOT assign to any well

SINGLE-WELL PROGRAMS:
- One well with many pages (50-150)
- Group pages by section: cover, casing, BHA, directional, fluids, risks
- Assign ALL technical pages to the one well
